# 飞书话题会话使用说明

本文说明 `codex-feishu` 在飞书话题（thread）场景下的会话行为。

## 1. 话题会话的绑定规则

- 会话键为：`用户 + chat + threadKey`。
- 在群聊或私聊里新开话题后，若该话题还没有绑定记录，会自动继承同一 `chat` 主会话（`threadKey = null`）的：
  - `agent` 绑定
  - `active thread` 绑定

这意味着你在主会话中绑定过一次 agent 后，在新话题里通常不需要再次执行 `/bind`。

## 2. 典型使用流程

1. 在主会话中执行一次 `/bind <agent>`。
2. 在同一 chat 新开飞书话题并 @ 机器人发送普通文本或 `/status`。
3. 机器人会在话题里直接续用继承的 agent。
4. 若你希望该话题切到其他 agent，可在话题内执行 `/bind <agent>` 覆盖当前话题绑定。

## 3. `/sessions` 会话列表展示

- `/sessions` 卡片按最近更新时间展示会话。
- 列表主信息优先展示摘要（线程名、最近用户消息、最近助手消息或 preview）。
- 线程 ID 仍用于按钮切换，但不作为主展示文案。

## 4. 故障排查

若在话题里仍提示“当前未绑定”：

1. 先在主会话执行 `/status`，确认主会话已绑定 agent。
2. 再回到话题执行 `/status` 触发继承绑定。
3. 如果服务刚更新，确认进程已重启并加载最新 `dist`。
